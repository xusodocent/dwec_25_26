    Exercici 1: Creació de repositoris locals i remots

    1. Crear un repositori local i remot:

    • Crea una nova carpeta local anomenada libro i inicialitza-hi un repositori Git:
    Resposta
    Bash
    1
2
3
       mkdir libro
   cd libro
   git init
    • Crea un repositori remot a GitHub amb el nom libro. Un cop creat, enllaça'l amb el teu repositori local:
    Resposta
    Bash
    1
    git remote add origin https://github.com/tuusuariodegithub/libro.git

    2. Configurar Git:

    • Defineix el teu nom d'usuari, el correu electrònic i activa el colorejat de la sortida de Git. Mostra la configuració final:
    Resposta
    Bash
    1
2
3
4
    git config --global user.name "El teu nom"
git config --global user.email "el.teu.email@example.com"
git config --global color.ui auto
git config --list

    3. Comprovar l'estat del repositori:

    • Comprova l'estat del repositori amb git status. Després, crea un fitxer anomenat indice.txt amb el següent contingut:

      Text Only
      1
2
3
      Capítol 1: Introducció a Git
Capítol 2: Flux de treball bàsic
Capítol 3: Repositoris remots

    • Afegeix el fitxer a l'staging area i comprova l'estat del repositori:

    Resposta
    Bash
    1
2
3
    git status
git add indice.txt
git status

    4. Fer un commit:

    • Realitza un commit dels canvis amb el missatge "Afegit índex del llibre":
    Resposta
    Bash
    1
    git commit -m "Afegit índex del llibre"

    5. Modificar fitxers i fer un commit addicional:

    • Modifica el fitxer indice.txt per afegir el text "Capítol 4: Gestió de branques". Realitza un commit amb el missatge "Afegit capítol 4 sobre gestió de branques":
    Resposta
    Bash
    1
2
    git add indice.txt
git commit -m "Afegit capítol 4 sobre gestió de branques"

    6. Pujar els canvis al repositori remot:

    • Puja els canvis al repositori remot a GitHub:
    Resposta
    Bash
    1
    git push -u origin main

    Exercici 2: Gestió de canvis

    1. Clonar el teu repositori remot:

    • Clona el repositori remot que has creat en l'Exercici 1:
    Resposta
    Bash
    1
    git clone https://github.com/tuusuariodegithub/libro.git

    2. Afegir fitxers nous i fer commits:

    • Crea la carpeta capitulos i el fitxer capitulo1.txt amb algun text. Afegeix els canvis a l'staging area i realitza un commit amb el missatge "Afegit capítol 1":
    Resposta
    Bash
    1
2
3
4
    mkdir capitulos
echo "Text del capítol 1" > capitulos/capitulo1.txt
git add capitulos/capitulo1.txt
git commit -m "Afegit capítol 1"

    3. Realitzar més canvis i comparar versions:

    • Crea el fitxer capitulo2.txt a la carpeta capitulos, fes un commit amb el missatge "Afegit capítol 2" i mostra les diferències amb la primera versió:
    Resposta
    Bash
    1
2
3
4
    echo "Text del capítol 2" > capitulos/capitulo2.txt
git add capitulos/capitulo2.txt
git commit -m "Afegit capítol 2"
git diff HEAD~1 HEAD

    Exercici 3: Col·laboració en repositoris remots

    1. Clonar el repositori remot https://github.com/xusodocent/practica_git:

    • Clona el repositori remot proporcionat per al treball col·laboratiu:
    Resposta
    Bash
    1
    git clone https://github.com/xusodocent/practica_git.git

    2. Afegir el fitxer autores.txt i pujar els canvis:

    • Crea un fitxer autores.txt amb el teu nom i correu electrònic. Afegeix-lo a l'staging area, fes un commit amb el missatge "Afegit autor" i puja els canvis:
    Resposta
    Bash
    1
2
3
4
    echo "Nom: El teu nom\nCorreu: el.teu.email@example.com" > autores.txt
git add autores.txt
git commit -m "Afegit autor"
git push origin main

    3. Bifurcació i col·laboració amb Pull Request:

    • Fes un "fork" del repositori https://github.com/xusodocent/practica_git al teu compte de GitHub.

    • Clona el repositori bifurcat al teu ordinador, crea una nova branca autoria i afegeix el teu nom i correu a autores.txt. Fes un commit amb el missatge "Afegit nou autor" i puja els canvis a la branca autoria al teu repositori remot:

    Resposta
    Bash
    1
2
3
4
5
6
    git clone https://github.com/tuusuariodegithub/practica_git.git
git checkout -b autoria
echo "Nom: El teu nom\nCorreu: el.teu.email@example.com" >> autores.txt
git add autores.txt
git commit -m "Afegit nou autor"
git push origin autoria
    • Canvia a la branca principal i fusiona amb auditoria:
    Resposta
    Bash
    1
2
    git checkout main
git merge bibliografia
    • Mostra l'historial:
    Resposta
    Bash
    1
    git log --oneline --graph --all
    • Finalment, crea un Pull Request al repositori original xusodocent/practica_git.

    Exercici 4: desfer canvis i tornar a l'estat anterior

    Per tornar a una versió anterior amb Git, hi ha diverses opcions depenent del que vols aconseguir.

    1. Desfer l'últim commit, però mantenint els canvis al directori de treball

    Si has fet un commit erroni però vols mantenir els canvis al directori per editar-los o corregir-los abans de fer un nou commit:

    • Exemple:

    Suposem que has fet un commit amb el missatge "Afegit capítol 5" i després t'adones que has comès un error. Per desfer aquest commit i mantenir els canvis, utilitza:

    Resposta
    Bash
    1
    git reset --soft HEAD^

    Això farà que l'últim commit desaparegui, però mantindrà els canvis en l'staging area, de manera que podràs fer un nou commit corregit.

    2. Desfer l'últim commit i eliminar també els canvis del directori de treball

    Si vols desfer un commit i també els canvis al fitxer, tornant el directori de treball exactament a l'estat en què estava abans del commit:

    • Exemple:

    Suposem que has fet un commit amb el missatge "Afegit capítol 5" però ja no vols aquests canvis. Per desfer el commit i també els canvis al fitxer, fes servir:

    Resposta
    Bash
    1
    git reset --hard HEAD^

    Això eliminarà tant el commit com els canvis que havies fet als fitxers. Tot el que s'havia afegit des de l'últim commit desapareixerà.

    3. Tornar a una versió anterior específica

    Si vols tornar a una versió anterior específica del teu projecte sense perdre l'historial (de manera que encara pots recuperar els commits després si ho necessites), pots utilitzar la comanda git checkout.

    • Exemple:

    Suposem que vols tornar al commit amb l'identificador abc1234. Pots fer-ho amb la comanda:

    Resposta
    Bash
    1
    git checkout abc1234

    Això et canviarà al commit abc1234, però en mode "detached HEAD", que significa que estàs en una revisió concreta i no en cap branca.

    4. Tornar a una versió anterior i mantenir els commits (amb revert)

    Si t'adones que un commit anterior tenia un error i vols tornar a aquell estat, però sense perdre els canvis posteriors (és a dir, fer un "revert"), pots utilitzar la comanda git revert.

    • Exemple:

    Si el commit erroni té l'ID abc1234, utilitza:

    Resposta
    Bash
    1
    git revert abc1234

    Això farà un nou commit que desfà els canvis introduïts per aquell commit, sense eliminar l'historial.

    • Exemple pràctic:

    Imagina que has fet aquests commits: 1. Commit inicial 2. Afegit capítol 1 3. Afegit capítol 2 (amb un error) 4. Afegit capítol 3

    I t'adones que el commit "Afegit capítol 2" és incorrecte.

    1. Per tornar al commit anterior (desfer els canvis del capítol 2) mantenint l'historial, pots fer:
    Resposta

    Bash
    1
    git revert <ID_commit_afegit_capitol_2>
    Això crearà un nou commit que eliminarà els canvis del capítol 2.

    1. Si en lloc d'això vols esborrar totalment els canvis des del commit "Afegit capítol 2" fins ara, pots utilitzar:
    Resposta
    Bash
    1
    git reset --hard <ID_commit_inicial>
