    Descripció del projecte

    Anem a crear una aplicació que simula els assistents a una reunió, de manera que es podrà afegir assistens i contar quants assistents hi han.

    Sense API

    Crearem el component AssistantsList.vue que té un array amb els assistents, juntament amb dos botons, un per a afegir un assistent i altre per a eliminar-lo. En cas d'afegir es crea un nom aleatori.

    <template>

    <div>
        <ul v-for="person in peopleList" :key="person">
            <li>{{person}}</li>
        </ul>
        <div>
            <button @click="addAssistant()">Afegir Assistent</button>
            <button @click="deleteAssistant()">Eliminar Assistent</button>
        </div>    
    </div>

</template>

<script>
export default {

    data: function (){
        return{
            peopleList:['Ana', 'Xuso', 'Pepe', 'Evaristo', 'Aitana'],
        }
    },
    methods: {
        addAssistant: function(){
            var randomNumber = Math.floor(Math.random() * 10001);
            this.peopleList.push(`Assistent${randomNumber}`);
        },
        deleteAssistant: function(){
            this.peopleList.splice(0,1);
        }
    }

}
</script>

    Necessitat de la gestió d'estats

    Per poder mantenir un comptador d'assistents a la sala, crearem un component que siga capaç de comptar el nombre d'assistents.

    Aquest és el punt crític en què els components AssistantsList.vue i el nou component (que anomenarem AssistantsCount.vue) han de transmetre's informació sobre el nombre d'assistents a la sala.

    En un enfocament tradicional, ens trobaríem amb el trasllat de variables entre components pares i fills mitjançant les props. A més, aquests components han d'emetre esdeveniments perquè el component pare faci un canvi en el valor d'aquestes variables.

    Quan només tenim un component pare i un fill, aquestes accions poden ser senzilles, però, què passa quan tenim molts components i la informació ha d'intercanviar-se entre ells? En aquests casos, entra en acció la 'font de veritat' de Vue, Vuex. Un contenidor que emmagatzemarà tota la informació que necessitem per l'intercanvi de dades entre components.

    En el nostre cas, el comptador d'assistents serà la variable que contindrà Vuex al contenidor principal (anomenat store). Aquestes variables s'anomenen estats.

    import { createStore } from 'vuex';

// Creem el store
const store = createStore({
  state() {
    return {
      // Aquí es defineix l'estat global. En aquest cas, un comptador.
      counter: 5
    };
  },
  mutations: {
    // Mutació per afegir un assistent (incrementar el comptador)
    ADD_PERSON(state) {
      state.counter++; // Incrementa el comptador
    },
    // Mutació per eliminar un assistent (decrementar el comptador)
    DELETE_PERSON(state) {
      state.counter--; // Decrementa el comptador
    }
  }
});

export default store;

    Explicació del codi:

    1. State: Aquesta és la part del store on guardem l'estat de l'aplicació. En aquest cas, tenim una variable counter inicialitzada a 5, que representa el nombre d'assistents.

    2. Mutations: Les mutacions són funcions que permeten modificar l'estat de manera síncrona. En aquest cas, tenim dues mutacions:

    3. ADD_PERSON: Incrementa el comptador (state.counter++).

    4. DELETE_PERSON: Decrementa el comptador (state.counter--).

    Els components de l'aplicació podran cridar aquestes mutacions per actualitzar l'estat del comptador.

    Modificar l'estat

    Finalment, només ens queda la possibilitat de modificar el nostre estat de comptador des de la llista on s'afegeixen i eliminen assistents.

    Un altre dels helpers que implementa Vuex és mutations (mutacions en valencià), que ens ajudarà a modificar el valor d'un estat. La declaració de les mutacions es realitza a /src/store.js i, en la seva declaració, es recomana utilitzar una nomenclatura de constants.

    En aquest cas, crearem una mutació que sumi un al nostre comptador per afegir un assistent a la sala, i una altra que realitzi la mateixa acció però restant el nombre d'assistents.

    Cal tindre en compte que a la funció que executa la mutació cal passar-li com a paràmetre l'estat del store per poder utilitzar els estats dins de la mateixa.

    Per a fer un commit utilitzem

    JavaScript
    1
    this.$store.commit([nombreMutación])
