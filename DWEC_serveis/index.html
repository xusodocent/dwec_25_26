    Ús de serveis

    Com organitzar les crides a una API com a servei en Vue 3

    Organitzar les crides a una API com a servei és una bona pràctica per mantenir el codi net, modular i fàcil de mantenir. Aquesta estratègia separa la lògica de les crides a l'API del component, millorant la reutilització i el testatge.

    Una estructura típica per organitzar els serveis seria així:

    import axios from 'axios';

// Configuració del client HTTP
const httpClient = axios.create({
  baseURL: 'http://localhost:3000', // URL base de la teua API
  timeout: 5000,                    // Temps màxim per a una crida
  headers: {
    'Content-Type': 'application/json',
  },
});

// Interceptors (opcional): pots afegir lògica per manejar errors o autenticació
httpClient.interceptors.response.use(
  response => response,
  error => {
    console.error('Error en la resposta:', error);
    return Promise.reject(error);
  }
);

export default httpClient;

    import httpClient from './httpClient';

const endpoint = '/courses'; // Endpoint específic de cursos

export const getAllCourses = async () => {
  const response = await httpClient.get(endpoint);
  return response.data; // Retorna les dades de la resposta
};

export const getCourseById = async (id) => {
  const response = await httpClient.get(`${endpoint}/${id}`);
  return response.data;
};

export const createCourse = async (course) => {
  const response = await httpClient.post(endpoint, course);
  return response.data;
};

export const updateCourse = async (id, course) => {
  const response = await httpClient.put(`${endpoint}/${id}`, course);
  return response.data;
};

export const deleteCourse = async (id) => {
  const response = await httpClient.delete(`${endpoint}/${id}`);
  return response.data;
};

    <template>
  <div>
    <h1>Llistat de cursos</h1>
    <ul>
      <li v-for="course in courses" :key="course.id">
        {{ course.name }} - {{ course.duration }}
        <button @click="deleteCourse(course.id)">Esborrar</button>
      </li>
    </ul>
  </div>
</template>

<script>
import { getAllCourses, deleteCourse } from '../api/coursesService';

export default {
  name: 'CourseList',
  data() {
    return {
      courses: [],
    };
  },
  async created() {
    // Carrega els cursos quan es monte el component
    this.courses = await getAllCourses();
  },
  methods: {
    async deleteCourse(id) {
      try {
        await deleteCourse(id);
        this.courses = this.courses.filter(course => course.id !== id);
      } catch (error) {
        console.error('Error esborrant el curs:', error);
      }
    },
  },
};
</script>

    <template>
  <div>
    <h1>{{ isEditing ? 'Editar curs' : 'Afegir curs' }}</h1>
    <form @submit.prevent="submitForm">
      <input v-model="course.name" placeholder="Nom del curs" required />
      <input v-model="course.duration" placeholder="Duració" required />
      <button type="submit">Guardar</button>
    </form>
  </div>
</template>

<script>
import { createCourse, updateCourse, getCourseById } from '../api/coursesService';

export default {
  name: 'CourseForm',
  props: {
    courseId: {
      type: Number,
      default: null, // Si és `null`, estem creant un nou curs
    },
  },
  data() {
    return {
      course: {
        name: '',
        duration: '',
      },
    };
  },
  computed: {
    isEditing() {
      return this.courseId !== null;
    },
  },
  async created() {
    // Si estem editant, carrega les dades del curs
    if (this.isEditing) {
      this.course = await getCourseById(this.courseId);
    }
  },
  methods: {
    async submitForm() {
      try {
        if (this.isEditing) {
          await updateCourse(this.courseId, this.course);
        } else {
          await createCourse(this.course);
        }
        alert('Curs guardat amb èxit!');
        this.$router.push('/');
      } catch (error) {
        console.error('Error guardant el curs:', error);
      }
    },
  },
};
</script>

    4. Beneficis d'aquesta organització

    1. Reutilització del codi:

    2. Les crides a l'API estan centralitzades en fitxers de servei, evitant duplicacions en els components.

    3. Mantenibilitat:

    4. Si l'endpoint de l'API canvia, només has de modificar-lo al servei.

    5. Testabilitat:

    6. Les funcions dels serveis poden ser testejades de manera independent dels components.

    7. Components més nets:

    8. Els components contenen només la lògica de presentació, mentre que les crides a l'API estan encapsulades en els serveis.
